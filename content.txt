1. In Python OOP, **`self` represents the current object (instance) of a class.**

It is used to access the variables and methods that belong to that particular object.

---

## ðŸ”¹ Why do we need `self`?

When we create a class, it is just a blueprint.
When we create an object from that class, each object has its own data.

`self` helps Python understand:

> â€œThis variable belongs to THIS specific object.â€

---

## ðŸ”¹ Basic Example

```python
class Student:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def introduce(self):
        print("Hi, my name is", self.name)

s1 = Student("Atharva", 20)
s2 = Student("Rahul", 22)

s1.introduce()
s2.introduce()
```

### ðŸ”Ž What is happening?

When you write:

```python
s1.introduce()
```

Python internally converts it to:

```python
Student.introduce(s1)
```

So:

* `self` = `s1` (first object)
* `self.name` = `"Atharva"`

Similarly:

* `self` = `s2` for second object
* `self.name` = `"Rahul"`

ðŸ‘‰ So `self` refers to the object that is calling the method.

---

## ðŸ”¹ Important Points About `self`

### 1ï¸âƒ£ It is NOT a keyword

`self` is just a naming convention.

You could technically write:

```python
def __init__(myobject, name):
    myobject.name = name
```

But by convention, we always use `self`.

---

### 2ï¸âƒ£ It must be the first parameter in instance methods

```python
def method(self):
```

Because Python automatically passes the calling object.

---

### 3ï¸âƒ£ Used to differentiate instance variables from local variables

Wrong:

```python
class Test:
    def __init__(self, x):
        x = x   # useless
```

Correct:

```python
class Test:
    def __init__(self, x):
        self.x = x
```

Here:

* Left side â†’ instance variable
* Right side â†’ parameter
